# Получение лимитов

## Назначение

Документ описывает варианты получения usage/limits по AI-провайдерам и служит точкой входа в provider-specific документацию.

Цель продукта - локально показать пользователю актуальные лимиты без лишней настройки. При этом у разных провайдеров данные доступны через разные каналы, поэтому для каждого провайдера фиксируются несколько возможных способов: основной, fallback и исследовательские варианты.

## Термины

- Usage - текущий расход за период.
- Limit - доступный лимит тарифа или включенного usage.
- Reset - момент сброса периода.
- Quota/rate limit - технические ограничения API или сервиса.
- Provider method - конкретный канал получения данных для провайдера.

## Варианты получения данных

| Вариант | Статус | Плюсы | Минусы | Провайдеры |
|---|---|---|---|---|
| Официальный API | Предпочтительно, если доступен | Стабильность, понятная поддержка, низкие риски | Часто требует ключ, включение API или Enterprise-тариф | Cursor Enterprise; потенциально Codex/Claude/API-провайдеры |
| Локальные transcript/telemetry файлы | Кандидат для MVP usage history | Быстро, локально, без сетевых запросов и без расхода лимитов | Обычно дает usage history, но не всегда официальный остаток лимита/reset | Claude (`~/.config/claude/projects`, `~/.claude/projects`, Xcode ClaudeAgentConfig), Codex (`~/.codex`), Gemini (`~/.gemini/tmp`) |
| Statusline/hook stdin | Кандидат для live limits | Может дать официальный live-сигнал (`rate_limits`) без парсинга TUI | Работает только внутри поддерживаемого hook-контекста; требует настройки | Claude Code |
| Локальная derived DB/cache | Вспомогательный слой | Ускоряет dashboard, инкрементальные пересчеты и историю после upstream cleanup | Не первичный источник истины; нужна синхронизация с исходными файлами | Claude (`~/.claude/usage.db`, tool-specific caches) |
| CLI провайдера | Используется в PoC | Работает в минимальном пользовательском сценарии, использует уже настроенный CLI | Медленно, хрупкий TUI-парсинг, запрос может расходовать ресурсы | Codex, Claude; Cursor частично |
| Локальный токен/клиентский backend | Используется в PoC для Cursor | Может работать без отдельного API-ключа, использует существующий login | Неофициальный контракт, возможны изменения, нужен аккуратный security review | Cursor |
| Frontend/dashboard API через cookie | Research-only | Часто дает те же данные, что веб-интерфейс | Cookie - чувствительный секрет, высокий риск безопасности и ToS | Потенциально Codex, Claude, Cursor |
| Наблюдение трафика | Research-only | Может помочь понять внутренние контракты | HTTPS, certificate pinning, высокий риск хрупкости и злоупотребления | Потенциально все |

## Принципы выбора метода

- Сначала использовать официальный и документированный способ, если он доступен пользователю без существенного ухудшения UX.
- Для минимального сценария отдавать приоритет уже установленному и авторизованному локальному инструменту.
- Не извлекать cookie, session token или refresh token без явного согласия пользователя и отдельной модели угроз.
- Не считать неофициальные endpoint'ы стабильным публичным контрактом.
- Для каждого provider method фиксировать качество данных: какие поля доступны, насколько они точны, есть ли reset и как часто можно обновлять данные.
- Реализовывать fallback только если он улучшает пользовательский сценарий и не повышает непропорционально security/ToS риск.
